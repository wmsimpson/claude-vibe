# Gaming Industry Patterns

Comprehensive data model and generation patterns for gaming login event demos.

## Data Model Overview

```
┌─────────────────────────────────┐
│        Login Events             │
│   (massive volume, 4.5M rows)  │
└─────────────────────────────────┘
```

Single-table model representing user login telemetry for an online gaming platform.
Every row is one successful login event enriched with device fingerprinting, GeoIP
data, and session correlation identifiers.

## Key Patterns

### Hash-Based Deterministic ID Generation

The generator relies heavily on `baseColumnType="hash"` for deterministic, repeatable
ID generation. Internal long columns are created with hash-based uniqueness, then
formatted as 32-character hex strings via `format="0x%032x"`.

```python
.withColumn("internal_ACCOUNTID", LongType(), minValue=0x1000000000000,
            uniqueValues=n_users, omit=True, baseColumnType="hash")
.withColumn("ACCOUNTID", StringType(), format="0x%032x", baseColumn="internal_ACCOUNTID")
```

> **Tier 3 Only:** `baseColumnType="hash"` uses internal UDFs that do NOT work over
> Databricks Connect. This generator is notebook-only.

### Country-Weighted Distribution

Country assignment uses weighted random values (60% USA, 20% UK, 20% AUS), seeded
from the ACCOUNTID to give each account a stable country:

```python
.withColumn("country", StringType(), values=["USA", "UK", "AUS"],
            weights=[0.6, 0.2, 0.2], baseColumn="ACCOUNTID", random=True)
```

### Derived Geography via CASE Expressions

City, ISP, coordinates, country code, and region are all derived from the `country`
column using nested CASE expressions with deterministic `pmod(abs(hash(...)))` bucketing:

- **USA:** New York / San Francisco / Chicago (3-way hash bucket on ACCOUNTID)
- **UK:** London (single value)
- **AUS:** Sydney (single value)
- **ISP:** 5 providers per country selected by hash bucket
- **Coordinates:** City-level lat/lng (e.g., New York = 40.7128, -74.0060)

### SHA256 Session and Client IDs

SESSION_ID and CLIENTID are deterministic SHA-256 hashes of concatenated parent columns:

```python
.withColumn("CLIENTID", StringType(), expr="sha2(concat(ACCOUNTID, CLIENTNAME), 256)")
.withColumn("SESSION_ID", StringType(), expr="sha2(concat(ACCOUNTID, CLIENTID), 256)")
```

### Client Name Assignment

CLIENTNAME is deterministically derived from DEVICEID using a hash-modulo selection
over four game client platforms:

```python
.withColumn("CLIENTNAME", StringType(), expr="""
    element_at(
        array('SwitchGameClient','XboxGameClient','PlaystationGameClient','PCGameClient'),
        (pmod(abs(hash(DEVICEID)), 4) + 1)
    )
""")
```

### IP Address from Bit Shifts

Internal IP addresses are stored as long integers and then converted to dotted-quad
notation using bit-shift arithmetic:

```python
.withColumn("REQUESTIPADDRESS", StringType(), expr="""
    concat(
        cast((internal_REQUESTIPADDRESS >> 24) & 255 as string), '.',
        cast((internal_REQUESTIPADDRESS >> 16) & 255 as string), '.',
        cast((internal_REQUESTIPADDRESS >> 8) & 255 as string), '.',
        cast(internal_REQUESTIPADDRESS & 255 as string)
    )
""")
```

### Post-Processing for Derived Time Columns

EVENT_HOUR and EVENT_DATE are not generated by dbldatagen; they are added as
post-processing transforms on the built DataFrame:

```python
df = spec.build()
df = (df
    .withColumn("EVENT_HOUR", hour(col("EVENT_TIMESTAMP")))
    .withColumn("EVENT_DATE", to_date(col("EVENT_TIMESTAMP")))
)
```

## Table Schema

### Login Events

| Column | Type | Description | Generation Pattern |
|--------|------|-------------|-------------------|
| `EVENT_TIMESTAMP` | TIMESTAMP | Login event time | Random in date range |
| `internal_ACCOUNTID` | LONG | Internal account key (omitted) | `baseColumnType="hash"`, uniqueValues=200K |
| `ACCOUNTID` | STRING | Public account ID (hex) | `format="0x%032x"`, baseColumn=internal_ACCOUNTID |
| `internal_DEVICEID` | LONG | Internal device key (omitted) | `baseColumnType="hash"`, baseColumn=internal_ACCOUNTID |
| `DEVICEID` | STRING | Public device ID (hex) | `format="0x%032x"`, baseColumn=internal_DEVICEID |
| `APP_VERSION` | STRING | Application version | Static: `"current"` |
| `AUTHMETHOD` | STRING | Authentication method | Values: `["OAuth", "password"]` |
| `CLIENTNAME` | STRING | Game client platform | CASE on DEVICEID hash: Switch/Xbox/Playstation/PC |
| `CLIENTID` | STRING | Client identifier | `sha2(concat(ACCOUNTID, CLIENTNAME), 256)` |
| `SESSION_ID` | STRING | Session correlation ID | `sha2(concat(ACCOUNTID, CLIENTID), 256)` |
| `country` | STRING | Player country | Weighted 60/20/20: USA/UK/AUS |
| `APPENV` | STRING | Application environment | Static: `"prod"` |
| `EVENT_TYPE` | STRING | Event classification | Static: `"account_login_success"` |
| `CITY` | STRING | GeoIP city | CASE on country: US cities / London / Sydney |
| `COUNTRY_CODE2` | STRING | ISO country code | CASE on country: US / UK / AU |
| `ISP` | STRING | Internet service provider | CASE on country: 5 ISPs per country |
| `LATITUDE` | DOUBLE | City latitude | CASE on CITY: actual coordinates |
| `LONGITUDE` | DOUBLE | City longitude | CASE on CITY: actual coordinates |
| `REGION_NAME` | STRING | State/province name | CASE on country + CITY |
| `internal_REQUESTIPADDRESS` | LONG | Internal IP key (omitted) | `baseColumnType="hash"`, baseColumn=internal_DEVICEID |
| `REQUESTIPADDRESS` | STRING | Dotted-quad IP address | Bit-shift expression on internal IP |
| `USERAGENT` | STRING | User agent string | `concat('Launch/1.0+', CLIENTNAME, ...)` |
| `EVENT_HOUR` | INTEGER | Hour of event (0-23) | Post-processing: `hour(EVENT_TIMESTAMP)` |
| `EVENT_DATE` | DATE | Date of event | Post-processing: `to_date(EVENT_TIMESTAMP)` |

```python
import dbldatagen as dg
from pyspark.sql.types import DoubleType, StringType, TimestampType, LongType
from pyspark.sql.functions import col, hour, to_date

ROW_COUNT = 4_500_000
NUMBER_OF_USERS = 200_000
NUMBER_OF_DEVICES = 250_000
NUMBER_OF_IPS = 40_000

START_TIMESTAMP = "2025-03-01 00:00:00"
END_TIMESTAMP = "2025-03-30 00:00:00"

spec = (
    dg.DataGenerator(spark, name="login_events", rows=ROW_COUNT, partitions=50, randomSeed=42)
    .withColumn("EVENT_TIMESTAMP", TimestampType(),
                data_range=dg.DateRange(START_TIMESTAMP, END_TIMESTAMP, "seconds=1"), random=True)
    .withColumn("internal_ACCOUNTID", LongType(), minValue=0x1000000000000,
                uniqueValues=NUMBER_OF_USERS, omit=True, baseColumnType="hash")
    .withColumn("ACCOUNTID", StringType(), format="0x%032x", baseColumn="internal_ACCOUNTID")
    .withColumn("internal_DEVICEID", LongType(), minValue=0x1000000000000,
                uniqueValues=NUMBER_OF_DEVICES, omit=True, baseColumnType="hash",
                baseColumn="internal_ACCOUNTID")
    .withColumn("DEVICEID", StringType(), format="0x%032x", baseColumn="internal_DEVICEID")
    .withColumn("APP_VERSION", StringType(), values=["current"])
    .withColumn("AUTHMETHOD", StringType(), values=["OAuth", "password"])
    .withColumn("CLIENTNAME", StringType(), expr="""
        element_at(
            array('SwitchGameClient','XboxGameClient','PlaystationGameClient','PCGameClient'),
            (pmod(abs(hash(DEVICEID)), 4) + 1)
        )
    """)
    .withColumn("CLIENTID", StringType(), expr="sha2(concat(ACCOUNTID, CLIENTNAME), 256)",
                baseColumn=["ACCOUNTID", "CLIENTNAME"])
    .withColumn("SESSION_ID", StringType(), expr="sha2(concat(ACCOUNTID, CLIENTID), 256)")
    .withColumn("country", StringType(), values=["USA", "UK", "AUS"],
                weights=[0.6, 0.2, 0.2], baseColumn="ACCOUNTID", random=True)
    .withColumn("APPENV", StringType(), values=["prod"])
    .withColumn("EVENT_TYPE", StringType(), values=["account_login_success"])
    .withColumn("CITY", StringType(), expr="""
        CASE
            WHEN country = 'USA' THEN element_at(
                array('New York', 'San Francisco', 'Chicago'),
                pmod(abs(hash(ACCOUNTID)), 3) + 1)
            WHEN country = 'UK' THEN 'London'
            WHEN country = 'AUS' THEN 'Sydney'
        END
    """, baseColumn=["country", "ACCOUNTID"])
    .withColumn("COUNTRY_CODE2", StringType(),
                expr="CASE WHEN country = 'USA' THEN 'US' WHEN country = 'UK' THEN 'UK' WHEN country = 'AUS' THEN 'AU' END",
                baseColumn=["country"])
    .withColumn("ISP", StringType(), expr="""
        CASE
            WHEN country = 'USA' THEN element_at(
                array('Comcast', 'AT&T', 'Verizon', 'Spectrum', 'Cox'),
                pmod(abs(hash(ACCOUNTID)), 5) + 1)
            WHEN country = 'UK' THEN element_at(
                array('BT', 'Sky', 'Virgin Media', 'TalkTalk', 'EE'),
                pmod(abs(hash(ACCOUNTID)), 5) + 1)
            WHEN country = 'AUS' THEN element_at(
                array('Telstra', 'Optus', 'TPG', 'Aussie Broadband', 'iiNet'),
                pmod(abs(hash(ACCOUNTID)), 5) + 1)
            ELSE 'Unknown ISP'
        END
    """, baseColumn=["country", "ACCOUNTID"])
    .withColumn("LATITUDE", DoubleType(), expr="""
        CASE
            WHEN CITY = 'New York' THEN 40.7128
            WHEN CITY = 'San Francisco' THEN 37.7749
            WHEN CITY = 'Chicago' THEN 41.8781
            WHEN CITY = 'London' THEN 51.5074
            WHEN CITY = 'Sydney' THEN -33.8688
            ELSE 0.0
        END
    """, baseColumn="CITY")
    .withColumn("LONGITUDE", DoubleType(), expr="""
        CASE
            WHEN CITY = 'New York' THEN -74.0060
            WHEN CITY = 'San Francisco' THEN -122.4194
            WHEN CITY = 'Chicago' THEN -87.6298
            WHEN CITY = 'London' THEN -0.1278
            WHEN CITY = 'Sydney' THEN 151.2093
            ELSE 0.0
        END
    """, baseColumn="CITY")
    .withColumn("REGION_NAME", StringType(), expr="""
        CASE
            WHEN country = 'USA' THEN
                CASE
                    WHEN CITY = 'New York' THEN 'New York'
                    WHEN CITY = 'San Francisco' THEN 'California'
                    WHEN CITY = 'Chicago' THEN 'Illinois'
                    ELSE 'Unknown'
                END
            WHEN country = 'UK' THEN 'England'
            WHEN country = 'AUS' THEN 'New South Wales'
            ELSE 'Unknown'
        END
    """, baseColumn=["country", "CITY"])
    .withColumn("internal_REQUESTIPADDRESS", LongType(), minValue=0x1000000000000,
                uniqueValues=NUMBER_OF_IPS, omit=True, baseColumnType="hash",
                baseColumn="internal_DEVICEID")
    .withColumn("REQUESTIPADDRESS", StringType(), expr="""
        concat(
            cast((internal_REQUESTIPADDRESS >> 24) & 255 as string), '.',
            cast((internal_REQUESTIPADDRESS >> 16) & 255 as string), '.',
            cast((internal_REQUESTIPADDRESS >> 8) & 255 as string), '.',
            cast(internal_REQUESTIPADDRESS & 255 as string)
        )
    """, baseColumn="internal_REQUESTIPADDRESS")
    .withColumn("USERAGENT", StringType(),
                expr="concat('Launch/1.0+', CLIENTNAME, '(', CLIENTNAME, '/)/', SESSION_ID)",
                baseColumn=["CLIENTNAME", "SESSION_ID"])
)

df = spec.build()
df = (df
    .withColumn("EVENT_HOUR", hour(col("EVENT_TIMESTAMP")))
    .withColumn("EVENT_DATE", to_date(col("EVENT_TIMESTAMP")))
)
```

## Common Demo Queries

### Login Volume by Hour

```sql
SELECT
    EVENT_HOUR,
    COUNT(*) AS login_count,
    COUNT(DISTINCT ACCOUNTID) AS unique_accounts
FROM login_events
GROUP BY EVENT_HOUR
ORDER BY EVENT_HOUR
```

### Device Distribution by Client

```sql
SELECT
    CLIENTNAME,
    COUNT(*) AS login_count,
    COUNT(DISTINCT DEVICEID) AS unique_devices,
    COUNT(DISTINCT ACCOUNTID) AS unique_accounts
FROM login_events
GROUP BY CLIENTNAME
ORDER BY login_count DESC
```

### Geographic Analysis

```sql
SELECT
    country,
    CITY,
    REGION_NAME,
    COUNT(*) AS login_count,
    COUNT(DISTINCT ACCOUNTID) AS unique_accounts,
    COUNT(DISTINCT ISP) AS isp_count
FROM login_events
GROUP BY country, CITY, REGION_NAME
ORDER BY login_count DESC
```

### ISP Breakdown by Country

```sql
SELECT
    country,
    ISP,
    COUNT(*) AS login_count,
    COUNT(DISTINCT REQUESTIPADDRESS) AS unique_ips
FROM login_events
GROUP BY country, ISP
ORDER BY country, login_count DESC
```

### Auth Method Usage

```sql
SELECT
    AUTHMETHOD,
    CLIENTNAME,
    COUNT(*) AS login_count,
    ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER (), 2) AS pct
FROM login_events
GROUP BY AUTHMETHOD, CLIENTNAME
ORDER BY login_count DESC
```

### Daily Login Trend

```sql
SELECT
    EVENT_DATE,
    COUNT(*) AS daily_logins,
    COUNT(DISTINCT ACCOUNTID) AS daily_active_users
FROM login_events
GROUP BY EVENT_DATE
ORDER BY EVENT_DATE
```
